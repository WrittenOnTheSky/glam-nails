# 1. OBJECTIVE

Создать сайт для маникюрного салона с ярким гламурным дизайном на русском языке. Сайт должен включать: главную страницу с услугами, прайс-лист, галерею работ, форму онлайн-записи на временной слот. Использовать подход JAMstack с отдельным backend API для обработки бронирований.

# 2. CONTEXT SUMMARY

### Архитектура проекта
- **Frontend**: Astro (статический генератор сайтов)
- **Backend**: Node.js + Express API сервер
- **База данных**: SQLite (локально, легко мигрирует на PostgreSQL)
- **Структура**: монорепозиторий с frontend/ и backend/ папками

### Целевая аудитория
- Женщины 18-45 лет, ищущие услуги маникюра
- Ценительницы красоты и ухода за собой

### Планируемый хостинг (для будущего)
- Frontend: Netlify / Vercel / GitHub Pages (бесплатно)
- Backend: Render.com (бесплатно до 750ч/месяц)

# 3. APPROACH OVERVIEW

### Стратегия
1. **Astro** как frontend — быстрая генерация статики, отличная производительность
2. **Клиентский рендеринг** для интерактивной формы записи (Astro island с React)
3. **REST API** на Express для управления бронированиями
4. **SQLite** для хранения данных — простота и portability

### Дизайн-система
- Цветовая палитра: розовый (#FF69B4), золотой (#FFD700), белый, темно-розовый
- Элегантная типографика с акцентом на заголовки
- Плавные анимации и hover-эффекты
- Современные карточки услуг с иконками

# 4. IMPLEMENTATION STEPS

### Фаза 1: Подготовка окружения
1. Инициализация проекта Astro
2. Настройка структуры папок (frontend/ + backend/)
3. Установка зависимостей (React для интерактивных компонентов)

### Фаза 2: Backend API
1. Настройка Express сервера
2. Инициализация SQLite базы данных
3. Создание API endpoints:
   - `GET /api/services` — список услуг
   - `GET /api/available-slots?date=YYYY-MM-DD` — свободные слоты
   - `POST /api/bookings` — создание бронирования
4. Валидация данных и обработка ошибок

### Фаза 3: Frontend - Основные страницы
1. **Главная страница** — hero-секция, краткий обзор услуг
2. **Услуги** — карточки с ценами и описанием
3. **Прайс-лист** — детальная таблица стоимости
4. **Галерея** — фотографии работ мастера (статичные изображения)
5. **Контакты** — адрес, телефон, время работы, карта

### Фаза 4: Система бронирования
1. Компонент выбора даты
2. Компонент выбора времени (слоты по 1 часу, 10:00-20:00)
3. Форма ввода данных клиента (имя, телефон)
4. Валидация и отправка на API
5. Экран подтверждения записи

### Фаза 5: Полировка и тестирование
1. Адаптивный дизайн (мобильные/десктоп)
2. Анимации и переходы
3. SEO-метатеги и Open Graph
4. Тестирование API endpoint'ов
5. End-to-end тестирование формы записи
